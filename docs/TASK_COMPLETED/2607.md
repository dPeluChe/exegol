## 2026-07-05: T127 — Progressive Disclosure Skills

Branch: `feat/t127-progressive-disclosure-skills`.

### Implementation
- `Skill` type (`packages/shared/src/types/skill.ts`) gained `always: boolean` — the
  escape hatch for tiny critical skills that should always be fully inlined.
- `skills/loader.ts` parses `always: true` from frontmatter (`meta.always`).
- `agents/spawn-context.ts`: new `buildSkillContext(projectPath, requestedNames?)` replaces
  the old full-body injection. `always: true` skills are always fully inlined
  (`## Skill: <name>\n\n<content>`) regardless of selection — that's the point of the
  escape hatch. Explicitly-selected non-always skills get a metadata-only pointer instead
  (`- **name**: description — read \`filePath\` when this task matches.`), ~100 tokens/skill
  instead of the full body. The CLI is expected to use its own read tool to load the file
  when the task matches.
- `buildSpawnContext` now always calls `buildSkillContext` (not gated on
  `config.skillNames?.length` — always-on skills need to show up even with no explicit
  selection).
